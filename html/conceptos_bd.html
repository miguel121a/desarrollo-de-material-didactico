Conceptos básicos de bases de datos

Conoce los fundamentos de las bases de datos, incluyendo conceptos esenciales, su estructura y los elementos que las conforman. Este tema te ayudará a comprender, de forma clara y sencilla, cómo se organiza la información antes de trabajar con bases de datos relacionales y no relacionales.

1

Conceptos esenciales

Antes de trabajar con una base de datos es importante conocer algunos términos básicos que se usan constantemente en su manejo. Estos conceptos aplican tanto a bases de datos relacionales como no relacionales, y son la base para entender cómo se organiza y se administra la información.

  • Dato

    Es la unidad más pequeña de información, por sí sola no dice mucho, por ejemplo el número "20" o el texto "María".

  • Información

    Es el resultado de organizar varios datos para que tengan un significado claro, por ejemplo "María tiene 20 años".

  • Sistema Gestor de Bases de Datos (SGBD)

    Es el software que se encarga de crear, guardar, modificar y proteger la información, tanto en bases de datos relacionales (como MySQL) como no relacionales (como MongoDB).

  • Base de datos relacional

    Guarda la información en tablas fijas, formadas por filas y columnas, donde cada dato debe encajar en un formato ya definido.

  • Base de datos no relacional

    Guarda la información de forma más libre, sin depender de tablas fijas, por ejemplo en documentos, lo que permite adaptarse más fácilmente a datos que cambian con frecuencia.

  • Esquema

    Es el diseño o "plano" de la base de datos: en el modelo relacional define las tablas y columnas; en el no relacional puede ser más flexible o incluso no existir de forma fija.

2

Estructura y elementos de una base de datos

La forma en que se organiza la información cambia dependiendo del tipo de base de datos. A continuación se explica cómo se estructura en cada caso, tanto en el modelo relacional, basado en tablas, como en el modelo no relacional, más flexible y adaptable a distintos formatos de información.

En una base de datos relacional

La información se organiza en tablas, muy parecido a una hoja de cálculo:

  • Tabla

    Es como una carpeta o una hoja donde se guarda toda la información relacionada con un mismo tema, por ejemplo "alumnos" o "productos". Está formada por filas y columnas, y su estructura debe estar definida desde antes de guardar datos en ella.

  • Registro (fila)

    Es un elemento específico dentro de la tabla, es decir, un solo caso, por ejemplo un alumno en particular llamado "Ana". Cada fila representa un registro completo con toda su información.

  • Campo (columna)

    Es cada una de las características que describen a un registro, por ejemplo el nombre, la edad o el correo electrónico. Todas las filas de la tabla deben tener las mismas columnas.

En una base de datos no relacional

La información se organiza de una forma más libre, generalmente en documentos:

  • Colección

    Es el equivalente a una tabla, es decir, un grupo donde se guardan documentos relacionados con un mismo tema, por ejemplo "alumnos". A diferencia de la tabla, no exige que todos los documentos tengan exactamente la misma forma.

  • Documento

    Es el equivalente a un registro: representa un solo caso, por ejemplo un alumno en particular. Se guarda en un formato parecido a JSON, donde cada documento puede tener campos distintos entre sí.

  • Campo (propiedad)

    Es cada una de las características que describen al documento, por ejemplo el nombre o la edad. A diferencia de las columnas de una tabla, los campos pueden variar de un documento a otro dentro de la misma colección.

Elementos que garantizan su correcto funcionamiento

Además de la estructura, existen elementos que ayudan a que la información se mantenga completa, relacionada correctamente y sea fácil de consultar. Algunos son propios de las bases de datos relacionales y otros aplican también a las no relacionales:

  • Clave primaria (Primary Key) — Bases de datos relacionales

    Es un dato que identifica a cada registro de forma única dentro de una tabla, como si fuera su "número de identificación". Gracias a ella, ningún registro se puede confundir con otro, incluso si comparten el mismo nombre u otros datos. En una base de datos no relacional existe un concepto equivalente, generalmente un identificador único generado automáticamente para cada documento.

  • Clave foránea (Foreign Key) — Bases de datos relacionales

    Es un campo que conecta la información de una tabla con la de otra, haciendo referencia a la clave primaria de esa segunda tabla. Esto permite relacionar, por ejemplo, un alumno con la escuela a la que pertenece. Las bases de datos no relacionales normalmente no usan claves foráneas, ya que suelen guardar la información relacionada dentro del mismo documento.

  • Índices — Ambos tipos de bases de datos

    Funcionan de manera parecida al índice de un libro: permiten que la base de datos encuentre la información mucho más rápido, en lugar de tener que revisar registro por registro o documento por documento cada vez que se hace una búsqueda. Tanto las bases de datos relacionales como las no relacionales pueden usar índices.

  • Restricciones (Constraints) — Principalmente en bases de datos relacionales

    Son reglas que se aplican a los campos de una tabla para evitar errores al guardar información, por ejemplo que un correo electrónico no se pueda repetir o que un campo obligatorio no pueda quedar vacío. En las bases de datos no relacionales estas reglas son opcionales y se conocen generalmente como reglas de validación.
