Configurar acceso seguro

Un desarrollador acaba de crear un clúster en MongoDB Atlas y necesita dejarlo listo para que su aplicación pueda conectarse de forma segura. Arrastra únicamente los pasos necesarios para lograrlo a la zona de respuesta.

Opciones

Crear un usuario de base de datos con usuario y contraseña
Exportar todos los documentos a un archivo JSON antes de conectarse
Agregar la dirección IP de la aplicación a la lista de acceso
Asignar el rol de administrador de clúster a todos los usuarios de la aplicación
Obtener la cadena de conexión desde el panel del clúster
Crear un índice en cada campo de la colección para mejorar la conexión
Verificar que el clúster tenga habilitado el cifrado en tránsito
Configurar un snapshot manual justo antes de realizar la primera conexión
