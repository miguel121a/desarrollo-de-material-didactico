Administración de bases de datos relacionales en la nube

Las bases de datos son un componente fundamental en las aplicaciones y páginas web, ya que almacenan y organizan la información necesaria para su funcionamiento. Debido a la importancia de los datos que contienen, es indispensable aplicar medidas de administración y seguridad que permitan proteger la información y garantizar su correcta gestión.

Una de las principales estrategias para administrar una base de datos consiste en crear usuarios y asignarles roles o privilegios específicos. Esto permite establecer distintos niveles de acceso según las funciones de cada persona, evitando que todos los usuarios tengan control total sobre la información almacenada.

El acceso a la base de datos comienza cuando un usuario proporciona sus credenciales. El sistema verifica su identidad y, una vez validado el acceso, determina los permisos asociados a su cuenta. De esta forma, cada usuario puede utilizar únicamente las funciones y recursos que le han sido autorizados.

La asignación de privilegios contribuye a proteger la información, ya que reduce el riesgo de accesos no autorizados, modificaciones indebidas o eliminación accidental de datos.

Privilegios más utilizados

Privilegio Descripción
SELECTPermite consultar y leer datos de una o varias tablas.
INSERTPermite agregar nuevos registros a las tablas de la base de datos.
UPDATEPermite modificar los registros existentes en las tablas de la base de datos.
DELETEPermite eliminar registros de las tablas de la base de datos.
CREATEPermite crear nuevos objetos, como bases de datos, tablas, vistas o índices.
DROPPermite eliminar objetos existentes, como bases de datos, tablas, vistas o índices.
ALTERPermite modificar la estructura de objetos existentes, como tablas o vistas.
INDEXPermite crear y gestionar índices en las tablas de la base de datos.
CREATE USERPermite crear nuevos usuarios en la base de datos.
GRANTPermite otorgar privilegios a otros usuarios para que puedan realizar acciones específicas en la base de datos.
REVOKEPermite revocar privilegios previamente otorgados a otros usuarios.
2

Ejemplos de permisos (GRANT)

A continuación se muestran ejemplos de comandos para otorgar distintos permisos sobre tablas, junto con su descripción.

Permiso Comando Descripción
Consultar (solo lectura) 
GRANT SELECT ON nombre_base_datos.nombre_tabla TO 'usuario'@'localhost';
 Permite visualizar la información almacenada en la tabla; no autoriza insertar, modificar ni eliminar registros.
Agregar registros 
GRANT INSERT ON nombre_base_datos.nombre_tabla TO 'usuario'@'localhost';
 Autoriza al usuario a insertar nuevos registros en la tabla, sin poder modificar o eliminar datos existentes.
Modificar registros 
GRANT UPDATE ON nombre_base_datos.nombre_tabla TO 'usuario'@'localhost';
 Permite actualizar registros ya almacenados en la tabla.
Eliminar registros 
GRANT DELETE ON nombre_base_datos.nombre_tabla TO 'usuario'@'localhost';
 Autoriza la eliminación de registros de la tabla.
Varios privilegios 
GRANT SELECT, INSERT, UPDATE ON nombre_base_datos.nombre_tabla TO 'usuario'@'localhost';
 Combina permisos para consultar, insertar y modificar datos (sin eliminar ni alterar la estructura).
Todos los privilegios 
GRANT ALL PRIVILEGES ON nombre_base_datos.nombre_tabla TO 'usuario'@'localhost';
 Otorga todos los privilegios sobre la tabla o base de datos; normalmente reservado para administradores.
Aplicar cambios 
FLUSH PRIVILEGES;
 Actualiza los privilegios cargados por el gestor de base de datos para que los cambios entren en vigor de inmediato.

Los privilegios pueden asignarse a usuarios o a roles. Usar roles facilita la administración de permisos; se crea un rol con los permisos requeridos y luego se asigna ese rol a los usuarios, evitando otorgar privilegios uno por uno.

Creación de roles y usuarios

  • Creación de rolCREATE ROLE 'nombre_rol';
  • Creación de usuarioCREATE USER 'usuario'@'host' IDENTIFIED BY 'contraseña';
  • Asignación de privilegios al rolGRANT SELECT, INSERT, UPDATE ON nombre_base_datos.nombre_tabla TO 'nombre_rol';
  • Asignación del rol a un usuarioGRANT nombre_rol TO 'usuario'@'host';
